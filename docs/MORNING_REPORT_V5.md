## 5. תמונות ראשיות גרועות — חלקי

**הבעיה**: כמה מוצרים מציגים תקריב של מדבקת Climacool / Adidas / patch מועדון במקום החולצה השלמה.

**מה שעשיתי**: כתבתי `scripts/audit-images.mjs` שעושה את הבא:

1. **HEAD-fetch על כל 3,111 התמונות** עם אותו טריק Referer שה-yupoo proxy משתמש בו (כך שזה עובד גם בלי `next dev` רץ)
2. **Concurrency=8 + 2 retries** לטיפול בכשלי רשת ארעיים — חשוב מאוד כי בריצה הראשונה הייתה התרעה רשת אמצעית שגרמה ל-661 false positives
3. **שתי בדיקות תקינות**:
   - אם image[0] שבור (404/קובץ ריק): מתעדים, אבל לא מסירים את המוצר אוטומטית (יותר מדי סיכון)
   - אם image[1] קטן בהרבה מ-image[0] (פחות מ-35% במשקל בייטים): כנראה תמונת תקריב/לוגו → מסירים את image[1] מהמוצר

**תוצאה אחרי retries**:
- 0 תמונות ראשיות שבורות (כל 1,646 המוצרים נטענים)
- 18 תמונות משניות (image[1]) הוסרו כתמונות תקריב

**מגבלה**: בלי computer vision אני לא יכול להבחין מה _בתוך_ התמונה. אם image[0] עצמה מציגה תקריב של מדבקה במקום החולצה — הסקריפט שלי לא ידע. הדרך היחידה לתקן את זה לחלוטין היא לעבור ידנית על המוצרים החשודים, או לחבר Google Vision / Claude vision API (אני יכול לבנות את זה אם תרצה).

**הצעה לבא**: אם תפתח `/products` ותסמן 5–10 מוצרים בעייתיים, אני אוסיף את ה-IDים שלהם לרשימת "needs-review" ואכין סקריפט שמסיר אותם בלחיצה.

## 6. סטים בלי מכנסיים — תוקן

**הבעיה**: מוצרים שסומנו `isShortSuit: true` אבל מציגים רק חולצה.

**הפתרון**: `scripts/fix-short-suit.mjs` שמבדל בין סטים אמיתיים לבין false positives:

**סיגנלים לסט אמיתי**:
- סיני: `套` / `套装` / `短裤` / `童装`
- עברית: `סט` / `חליפ` / `מכנס` / `מדים`
- אנגלית: `kit` / `set` / `suit` / `short`

**אם אין שום סיגנל** → הסטה היא false positive → מנקים `isShortSuit=false` ומסירים את ה-tags `סטים` ו-`short-suit`.

**תוצאה**:
- 410 מוצרים סימנו isShortSuit
- 281 עם סיגנלים מאומתים → נשמרים ✅
- **129 בלי סיגנל → נוקה**

עכשיו `/collections/short-suit` יראה 281 סטים אמיתיים בלבד.
